# MessageList.tsx 정밀 테스트 시나리오 (50개)

## [그룹 1: 가상화 리스트 및 성능] (1-10)
1. **초기 렌더링**: 메시지가 10개 미만일 때 가상화 없이 모든 메시지가 정상 표시되는가.
2. **대량 메시지**: 메시지가 500개 이상일 때 스크롤 시 끊김(Jank) 현상이 없는가.
3. **DOM 최적화**: 화면 밖의 메시지 요소가 실제 DOM에서 제거되는지 개발자 도구로 확인.
4. **오버스캔**: 화면 바로 위/아래 영역의 메시지를 미리 렌더링하여 빠른 스크롤 시 빈칸 노출 최소화.
5. **동적 높이 측정**: 이미지가 로드되어 메시지 높이가 변할 때 하단 메시지들이 밀려나지 않고 위치를 재보정하는가.
6. **창 크기 조절**: 브라우저 창 너비를 바꿀 때 `ResizeObserver`가 실시간으로 높이를 재측정하는가.
7. **메모리 유지**: 대화방 전환 시 이전 대화의 `measuredHeights` 캐시를 보존하여 복귀 시 빠른 렌더링 지원.
8. **화이트아웃 방지**: 이분 탐색(`findStartIndex`) 로직 오류로 인해 전체 목록이 사라지는 현상 방지.
9. **렌더링 Nonce**: `renderNonce` 변경 시 가상화 상태를 초기화하고 강제 재렌더링하는가.
10. **Fallback Viewport**: 부모 컨테이너 높이 측정 실패 시 기본값(`720px`)으로 안정적으로 렌더링되는가.

## [그룹 2: 스크롤 관리 및 UX] (11-25)
11. **위치 복원**: 대화 선택 시 `initialScrollTop` 위치로 부드럽게 이동하는가.
12. **자동 하단 고정**: 스트리밍 중 새 텍스트가 추가되면 스크롤이 바닥을 따라가는가.
13. **수동 중단 감지**: 사용자가 위로 30px 이상 올리면 즉시 `isAutoBottom`이 false가 되는가.
14. **추적 재개**: 수동 스크롤 후 다시 바닥에 도달하면 `isAutoBottom`이 true로 복구되는가.
15. **스크롤 떨림 방지**: 높이 측정과 스크롤 이동이 동시에 일어날 때 화면이 위아래로 진동하지 않는가.
16. **키보드 내비게이션**: PageUp/Down 키 사용 시 가상화 영역이 정상 업데이트되는가.
17. **마우스 휠**: 고해상도 휠 입력 시 스크롤 이벤트가 누락되지 않는가.
18. **터치 스크롤**: 모바일/태블릿 환경에서의 관성 스크롤 대응 여부.
19. **스크롤바 유무**: 메시지가 적어 스크롤바가 없을 때 하단 여백(`MESSAGE_LIST_BOTTOM_PADDING`)이 유지되는가.
20. **앵커 이동**: 특정 메시지 ID로 이동(`scrollToId`) 시 정확한 위치에 정지하는가.
21. **상단 여백**: 첫 번째 메시지 위의 간격이 디자인 명세와 일치하는가.
22. **하단 여백**: 마지막 메시지 아래의 스트리밍용 여백(320px)이 정확히 확보되는가.
23. **브라우저 히스토리**: 뒤로가기 시 이전 스크롤 위치가 캐시에서 복원되는가.
24. **다크모드 전환**: 테마 변경 시 배경색과 텍스트색이 즉시 반영되며 스크롤 위치가 유지되는가.
25. **빈 대화방**: 메시지가 하나도 없을 때 "대화 내용이 없습니다" 등의 플레이스홀더 표시 여부.

## [그룹 3: 마크다운 및 인터랙티브 콘텐츠] (26-35)
26. **코드 하이라이팅**: 프로그래밍 언어별 문법 강조가 정확히 적용되는가.
27. **코드 복사 버튼**: 코드 블록 우측 상단 버튼 클릭 시 클립보드에 정확히 복사되는가.
28. **이미지 뷰포트**: 대용량 이미지가 포함된 경우 뷰포트 근처에서만 로딩을 시작하는가.
29. **이미지 확대**: 이미지 클릭 시 전체 화면으로 크게 볼 수 있는 라이트박스 기능.
30. **수식 렌더링**: LaTeX 형식이 수학 공식으로 예쁘게 그려지는가.
31. **Mermaid 다이어그램**: 다이어그램 코드가 실제 그래프 이미지로 변환되어 표시되는가.
32. **인라인 링크**: 외부 링크 클릭 시 시스템 브라우저로 새 창이 열리는가.
33. **출처 카드(Citation)**: 인라인 출처 클릭 시 상세 정보 패널(SourceDrawer)이 열리는가.
34. **GFM 지원**: 체크박스, 테이블, 취소선 등 GitHub Flavored Markdown 문법 지원.
35. **HTML 보안**: 마크다운 내 위험한 스크립트 태그가 실행되지 않도록 Sanitization 처리.

## [그룹 4: 실시간 스트리밍 UI] (36-45)
36. **초기 플레이스홀더**: 메시지 수신 시작 전 점 세개(...) 등의 로딩 애니메이션 노출.
37. **깜박이는 커서**: 스트리밍 중인 텍스트 끝에 블링킹 커서(`::after`) 표시 여부.
38. **스트리밍 종료**: `__GPT_STREAM_DONE__` 수신 시 커서가 즉시 사라지는가.
39. **가로 스크롤 방지**: 긴 단어나 코드 라인이 스트리밍될 때 전체 레이아웃이 깨지지 않는가.
40. **실시간 출처 교체**: 텍스트 스트리밍 중 `cite` 토큰이 나타나면 즉시 링크 카드로 변환되는가.
41. **정보 지연 반영**: 본문이 먼저 나오고 출처 메타데이터가 늦게 올 때 링크 카드가 자연스럽게 업데이트되는가.
42. **중첩 괄호 제거**: `(【1】)` 주변의 괄호가 렌더링 시점에 시각적으로 제거되는가.
43. **모델 아이콘**: 답변 중인 모델(GPT-4, Claude 등)의 아이콘이 메시지 좌측에 정상 표시되는가.
44. **에러 메시지**: 스트리밍 중단 시 빨간색 에러 안내 문구 노출.
45. **중단 버튼**: 답변 생성 중 '중단' 클릭 시 스트리밍 UI가 즉시 정지 상태로 변하는가.

## [그룹 5: 목차(TOC) 및 내비게이션] (46-50)
46. **질문 추출**: 사용자 메시지의 첫 줄을 목차 제목으로 자동 등록하는가.
47. **헤딩 감지**: 어시스턴트 답변 내의 `#`, `##` 태그를 실시간으로 목차에 추가하는가.
48. **좌측 탭 호버**: 화면 왼쪽 끝 탭에 마우스 호버 시 목차 패널이 스무스하게 열리는가.
49. **섹션 점프**: 목차 아이템 클릭 시 해당 위치로 스크롤 애니메이션과 함께 이동하는가.
50. **반응형 목차**: 창 너비가 좁아질 때 목차가 콘텐츠를 가리지 않도록 자동 숨김 처리.

## [그룹 6: 접근성(A11y) 및 다국어] (51-60)
51. **키보드 포커스 트랩 방지**: 탭(Tab) 키로 메시지 내의 모든 링크와 버튼을 순차적으로 탐색할 수 있는가.
52. **스크린 리더(ARIA)**: 출처 카드나 토글 버튼의 상태(`aria-expanded` 등)가 스크린 리더에 정확히 읽히는가.
53. **고대비 모드(High Contrast)**: OS 고대비 모드 활성화 시 텍스트와 배경 간의 대비가 명확한가.
54. **글꼴 크기 조절**: 시스템 폰트 크기를 극단적으로 키웠을 때 레이아웃이 깨지지 않고 유동적으로 늘어나는가.
55. **RTL(Right-to-Left) 텍스트**: 아랍어, 히브리어 등 우측에서 좌측으로 읽는 텍스트가 정상 정렬되는가.
56. **시각적 깜박임 최소화**: `prefers-reduced-motion` 설정 시 커서 깜박임이나 목차 애니메이션이 중단되는가.
57. **대체 텍스트(Alt Text)**: 자동 생성된 이미지나 다이어그램에 적절한 `alt` 속성이 부여되는가.
58. **초점 가시성**: 버튼 클릭이나 탭 이동 시 아웃라인(focus ring)이 시각적으로 명확히 표시되는가.
59. **언어 속성**: 렌더링된 마크다운 내 다국어 코드 블록에 올바른 `lang` 속성이 들어가는가.
60. **오류 메시지 접근성**: 로딩 실패나 네트워크 에러 발생 시 시각 장애인도 이를 즉시 인지할 수 있는 ARIA Live Region 적용 여부.

## [그룹 7: 이벤트 및 인터랙션 충돌 방지] (61-70)
61. **텍스트 드래그 선택**: 목차 패널 근처에서 텍스트를 마우스로 드래그할 때 이벤트가 끊기지 않는가.
62. **더블 클릭 복사**: 코드 블록을 더블 클릭했을 때 코드만 선택되고 외부 UI 요소가 선택되지 않는가.
63. **스크롤 중 클릭 무시**: 스크롤바를 빠르게 드래그하는 도중에 실수로 눌린 링크가 작동하지 않도록 방지.
64. **드롭다운 중첩**: 출처 서랍(SourceDrawer)이 열린 상태에서 창 크기를 줄여도 서랍이 닫히거나 깨지지 않는가.
65. **모달과의 겹침(Z-Index)**: 설정 모달 등 전체 화면 오버레이가 떴을 때 메시지 리스트의 요소가 모달 위로 뚫고 나오지 않는가.
66. **이벤트 버블링**: 특정 링크 클릭 시 상위 메시지 버블의 선택 이벤트가 함께 트리거되지 않는가(stopPropagation).
67. **빠른 연타(Debouncing)**: 목차 아이템을 매우 빠르게 여러 번 클릭해도 스크롤 점프 로직이 꼬이지 않는가.
68. **키보드 단축키 충돌**: 복사(`Ctrl+C`) 등 기본 단축키가 포커스 상태와 무관하게 정상 작동하는가.
69. **터치 디바이스 호버 픽스**: 모바일 환경에서 호버(Hover) 스타일이 터치 후 고착(Sticky)되지 않는가.
70. **컨텍스트 메뉴**: 메시지 위에서 우클릭 시 브라우저 기본 컨텍스트 메뉴가 정상적으로 노출되는가.

## [그룹 8: 메모리 누수 및 리소스 관리] (71-80)
71. **Observer 해제**: `ResizeObserver` 및 `IntersectionObserver`가 컴포넌트 언마운트 시 즉각 해제되는가.
72. **이벤트 리스너 정리**: 글로벌 `window`나 `document`에 등록된 스크롤/키보드 이벤트 리스너의 클린업 검증.
73. **거대 DOM 트리 폐기**: 대화방을 완전히 빠져나갔을 때 이전 대화방의 수천 개 DOM 노드가 메모리에서 수거되는가.
74. **이미지 캐싱 제어**: 수백 개의 이미지를 스크롤하며 볼 때 브라우저 메모리 한도를 넘지 않게 지연 로딩(Lazy Loading) 되는가.
75. **상태 배열 참조 끊기**: `useMemo`나 `useEffect`의 의존성 배열 누락으로 인한 클로저 내 메모리 릭이 없는가.
76. **렌더링 캐시 무효화**: 테마 변경 시 이전 테마 색상으로 렌더링된 캔버스나 마크다운 캐시가 갱신되는가.
77. **애니메이션 프레임 정리**: `requestAnimationFrame`을 사용한 부드러운 스크롤 시 진행 중 언마운트되면 `cancelAnimationFrame`이 호출되는가.
78. **iframe 리소스 관리**: 렌더링된 다이어그램 등 iframe 리소스가 뷰포트를 벗어나면 정지/해제되는가.
79. **비동기 요청 취소**: 컴포넌트가 사라진 후에도 출처 썸네일을 요청하는 네트워크 통신이 취소(Abort)되는가.
80. **로컬 스토리지 한도**: 스크롤 위치나 높이 캐시 데이터가 지나치게 커져 `QuotaExceededError`를 유발하지 않는가.

## [그룹 9: 레이아웃 시프트 및 스타일 엣지 케이스] (81-90)
81. **글꼴 로딩 지연**: 커스텀 폰트(예: 나눔고딕, Fira Code)가 늦게 로딩될 때 발생하는 높이 변화(Layout Shift) 대응.
82. **비정상적 긴 문자열**: 공백 없는 1,000자 이상의 긴 단어가 들어왔을 때 레이아웃 밖으로 삐져나가지 않고 줄바꿈(`word-break`) 처리.
83. **연속된 빈 줄**: 마크다운 본문에 엔터(빈 줄)가 수십 개 연속으로 있을 때 여백이 비정상적으로 커지지 않게 압축(Collapse) 처리.
84. **극소형 뷰포트**: 브라우저 창 크기를 가로 300px 이하로 극단적으로 줄였을 때 UI 요소가 겹치지 않는가.
85. **극대형 디스플레이**: 4K/8K 해상도 풀스크린에서 메시지 박스 최대 너비(Max-width)가 적절히 제한되어 가독성을 유지하는가.
86. **동적 스크롤바 생성**: 콘텐츠 증가로 스크롤바가 갑자기 생길 때 너비 차이로 인한 가로 흔들림(`scrollbar-gutter`) 방지.
87. **플렉스박스 오버플로우**: Flex 컨테이너 내의 자식 요소 텍스트가 넘칠 때 부모 영역을 강제로 늘리지 않는가.
88. **안전 영역(Safe Area)**: 노치(Notch) 디스플레이 등 기기별 하드웨어 안전 영역과 겹치지 않게 여백이 들어가는가.
89. **시스템 다크모드 오버라이드**: OS 다크모드와 앱 내 라이트모드 설정이 충돌할 때 앱 내 설정이 우선시되는가.
90. **프린트 모드 스타일**: `Ctrl+P` 인쇄 시 배경색, 그림자, 스크롤바 등이 숨겨지고 텍스트 위주의 깔끔한 레이아웃으로 출력되는가.

## [그룹 10: 상태 동기화 및 생명주기] (91-100)
91. **props 지연 업데이트**: `activeConversation`이 변경되는 도중 이전 대화의 잔여 상태(높이 캐시 등)가 섞이지 않게 초기화되는가.
92. **의존성 배열 순환(Loop)**: `useEffect`나 `useCallback` 내에서 상호 참조로 인한 무한 렌더링 루프 방지 로직.
93. **동시 렌더링(Concurrent Mode) 안전성**: React 18+ 환경에서 렌더링 중단/재개 시 상태 불일치(Tearing)가 발생하지 않는가.
94. **Strict Mode 두 번 호출**: 개발 모드 `<StrictMode>`에서 마운트-언마운트-마운트 사이클을 거쳐도 버그나 메모리 릭이 없는가.
95. **Stale Closure 문제**: 비동기 콜백(예: `setTimeout`, `fetch`) 내부에서 참조하는 컴포넌트 상태값이 최신 버전을 가리키는가(`useRef` 활용).
96. **전역 상태 의존성**: 테마, 윈도우 크기 등 전역 Context 상태 변화 시 꼭 필요한 하위 컴포넌트만 재렌더링되는가.
97. **비동기 초기화**: 스크롤 캐시나 로컬 스토리지 데이터를 비동기로 불러오는 찰나의 시간 동안 깜빡임(Flicker) 방지.
98. **unmount 시점의 setState**: 컴포넌트가 파괴된 후 진행 중이던 비동기 응답이 도착해 `setState`를 호출하려는 시도(Warning) 차단.
99. **데이터 경합(Race Condition)**: 연달아 대화방 A -> B -> C 로 아주 빠르게 이동할 때 최종 렌더링 결과가 C와 정확히 일치하는가.
100. **초기 스크롤 타이밍**: DOM이 완전히 마운트되고 페인트 되기 직전(`useLayoutEffect`)에 `initialScrollTop`을 정확히 적용하는가.

## [그룹 11: 가상화 스크롤 복합 시나리오 (Advanced Virtualization)] (101-110)
101. **비대칭 메시지 높이**: 높이가 50px인 메시지와 5000px인 메시지가 번갈아 나타날 때 스크롤 점프나 위치 잃음이 발생하지 않는가.
102. **초고속 스크롤(Fling)**: 트랙패드를 세게 튕겨 순식간에 수천 픽셀을 이동할 때 브라우저 렌더링 스레드가 멈추지 않는가.
103. **역방향 스크롤 생성**: 위로 스크롤할 때 위에 있던 메시지들이 DOM에 추가되면서 현재 보고 있는 화면의 위치가 밀려나지 않는가(스크롤 앵커링).
104. **동시 다발적 높이 변경**: 화면 내 여러 이미지나 iframe이 동시에 로딩 완료되어 여러 번의 `onHeightChange`가 동시에 쏟아질 때의 처리.
105. **오버스캔 한계 초과**: `MESSAGE_LIST_OVERSCAN` 값(800px)보다 큰 단일 메시지가 있을 때 이분 탐색이 꼬이지 않는가.
106. **리스트 중간 진입**: URL 해시나 검색 결과 클릭으로 리스트의 한가운데(예: 500번째 메시지)로 바로 진입 시 정확한 인덱스 계산.
107. **Zoom In/Out**: 브라우저 확대/축소를 빠르게 여러 번 수행했을 때 `viewportHeight`와 `measuredHeights`가 깨지지 않는가.
108. **리사이즈 중 스크롤**: 창 크기를 마우스로 드래그하여 줄이는 동시에 휠 스크롤을 할 때 레이아웃 파괴 여부.
109. **메시지 삭제/필터링**: 중간에 있는 수십 개의 메시지가 일괄 삭제되거나 필터링되어 숨겨질 때 가상화 인덱스의 동기화.
110. **초기화 전 렌더링 방어**: `viewportHeight`가 0인 상태(첫 렌더 사이클)에서 모든 메시지가 렌더링되는 성능 폭주 방지.

## [그룹 12: 동적 높이 변화 및 리사이징 (Dynamic Heights)] (111-120)
111. **토글 버튼 클릭**: "더 보기"나 아코디언 컴포넌트를 클릭하여 메시지 높이가 동적으로 변할 때 즉각 반영되는가.
112. **늦게 로딩되는 수식**: MathJax/KaTeX 등 비동기 수식 렌더러가 로딩된 후 텍스트 줄바꿈이 변할 때의 높이 보정.
113. **플로팅 요소 간섭**: 메시지 내부에 `float: left` 등의 CSS가 사용되어 높이 계산을 방해할 때 `ResizeObserver`의 정확도.
114. **에러 바운더리 높이**: 마크다운 파싱 에러로 인해 ErrorBoundary의 Fallback UI가 떴을 때의 높이 측정.
115. **비표준 폰트 로드**: Web Font 로딩 완료 전후로 높이가 미세하게(1~2px) 변할 때 불필요한 무한 루프 렌더링 방지.
116. **코드 래핑 모드 토글**: 코드 블록의 `word-wrap` 설정(가로 스크롤 vs 줄바꿈)을 바꿀 때 높이 동기화.
117. **초기 추정치 오차**: `estimateMessageHeight`로 계산한 추정치와 실제 측정 높이의 차이가 클 때 스크롤 위치 보정 로직의 부드러움.
118. **TOC 패널 토글**: 창 너비 변경으로 TOC 패널이 생기거나 사라져서 본문 너비가 바뀔 때 전체 높이 재계산.
119. **ResizeObserver 루프 경고**: "ResizeObserver loop limit exceeded" 콘솔 에러를 적절히 우회하거나 억제하고 있는가.
120. **미세한 소수점 높이**: 브라우저에 따라 요소 높이가 소수점(`150.4px`)으로 떨어질 때 `Math.ceil` 처리의 정합성.

## [그룹 13: 컴포넌트 생명주기 및 메모리 (Lifecycle & Memory)] (121-130)
121. **빠른 대화방 전환**: 대화방을 1초에 5번씩 스위칭할 때 이전 대화방의 비동기 작업(fetch, measure)이 모두 취소되는가.
122. **언마운트 시 콜백 억제**: 컴포넌트 언마운트 직후 `onHeightChange`나 `onScrollPositionChange`가 호출되어 경고를 내뿜지 않는가.
123. **이벤트 리스너 중복 등록**: `useEffect` 내 이벤트 리스너(키보드 등)가 재렌더링 시 누적되어 다중 실행되지 않는가.
124. **가비지 컬렉션 지연**: 수천 개의 메시지 객체가 포함된 `activeConversation` 참조가 메모리 상에 길게 남아있지 않은지 검증.
125. **Ref 값 최신화**: 비동기 스트리밍 리스너 내부에서 `isAutoBottomRef` 등의 `ref.current`가 항상 최신 상태를 유지하는가.
126. **불변성 파괴 방지**: `measuredHeights` 등의 객체를 업데이트할 때 기존 객체를 직접 변형(mutate)하지 않는지 확인.
127. **React DevTools 프로파일링**: 프로파일러 측정 시 불필요한 노드 렌더링(Wasted Renders)이 5% 미만인지 확인.
128. **iframe 메모리 해제**: 가상화 밖으로 밀려난 다이어그램 iframe 노드의 메모리가 정상적으로 반환되는가.
129. **캐시 비우기**: 사용자가 '캐시 지우기' 기능을 실행했을 때 메모리와 DOM 상태가 즉시 초기화되는가.
130. **메모리 릭 테스트**: Chrome DevTools 메모리 탭에서 스크롤을 5분간 지속했을 때 힙(Heap) 메모리가 계단식으로 계속 증가하지 않는가.

## [그룹 14: 스트리밍 중단 및 에러 복구 (Streaming & Errors)] (131-140)
131. **일시적 네트워크 단절**: 스트리밍 중 와이파이가 끊겼을 때 로딩 UI가 유지되고, 연결 복구 시 폴백 로직이 도는가.
132. **서버 타임아웃 504**: 게이트웨이 타임아웃 에러 수신 시 스트리밍 커서가 멈추고 적절한 에러 문구가 표시되는가.
133. **강제 프로세스 종료**: 메인 프로세스가 재시작되었을 때 렌더러의 스트리밍 상태(`receiving`)가 `idle`로 안전하게 초기화되는가.
134. **잘못된 스트림 조각**: 마크다운 파서를 깨뜨리는 비정상적인 문자열 조각 수신 시 크래시 방지.
135. **중복된 DONE 신호**: `__GPT_STREAM_DONE__`이 여러 번 연속으로 들어와도 상태 업데이트가 꼬이지 않는가.
136. **대기 중 채널 변경**: 답변 생성 대기 상태에서 다른 대화방으로 넘어갔을 때, 답변이 도착하면 백그라운드에서 조용히 추가되는가.
137. **에러 후 재시도**: 에러 발생 후 '다시 생성(Regenerate)' 버튼 클릭 시 이전 에러 UI가 즉시 지워지고 재스트리밍 되는가.
138. **Rate Limit 도달**: 429 Too Many Requests 에러 시 안내 문구와 함께 스트리밍 애니메이션 즉시 정지.
139. **빈 텍스트 스트리밍 종료**: 아무 텍스트 없이 스트리밍이 정상 종료(`[DONE]`)되었을 때 UI에 잔여물이 남지 않는가.
140. **폴백 타이머 작동**: 스트리밍 조각이 30초 이상 오지 않을 때 `useSharedConversationActions`의 폴백 타이머가 정확히 동작하는가.

## [그룹 15: 목차(TOC) 동적 업데이트 (Dynamic TOC)] (141-150)
141. **새로운 헤딩 실시간 감지**: 스트리밍 중 `## 새로운 주제`가 생성되는 순간 TOC 패널에 항목이 즉시 추가되는가.
142. **TOC 중복 키 방지**: 동일한 이름의 헤딩이 여러 개 있을 때 TOC 리스트의 React `key`가 충돌하지 않는가.
143. **TOC 스크롤 동기화**: 스크롤을 내릴 때 현재 화면에 보이는 섹션이 TOC 패널에서 하이라이트(Active) 처리되는가.
144. **TOC 클릭 시 스크롤 위치**: TOC 항목 클릭 시 상단 여백(header offset)을 고려하여 내용이 가려지지 않게 정지하는가.
145. **TOC 내용 긴 경우**: 추출된 헤딩 텍스트가 100자를 넘을 때 TOC 패널 내에서 말줄임표(`...`)로 깔끔하게 잘리는가.
146. **스트리밍 중 TOC 클릭**: 스트리밍이 한창 진행 중일 때 TOC의 이전 섹션을 클릭하면 자동 스크롤(`AutoBottom`)이 즉시 해제되는가.
147. **빈 목차 처리**: 대화 내에 헤딩이나 사용자 질문이 아예 없을 때 TOC 탭이 숨겨지거나 비활성화되는가.
148. **마크다운 특수문자 TOC**: `# **굵은 제목**`과 같은 마크다운이 TOC 텍스트로 추출될 때 마크다운 문법이 제거되고 순수 텍스트만 표시되는가.
149. **TOC 렌더링 성능**: 목차 항목이 100개 이상일 때 본문 타이핑 속도에 지연(Lag)을 주지 않도록 Memoization이 잘 되어 있는가.
150. **키보드 접근(TOC)**: Tab 키를 통해 TOC 리스트로 포커스를 이동하고 Enter로 점프할 수 있는가.

## [그룹 16: 마크다운 렌더링 엣지 케이스 (Markdown Edge Cases)] (151-160)
151. **지원되지 않는 언어**: ````unknown_lang` 형식의 코드 블록이 들어올 때 일반 텍스트 래퍼로 안전하게 폴백되는가.
152. **닫히지 않은 코드 블록**: 백틱 3개로 시작한 후 스트리밍이 끊겨 닫히지 않았을 때 전체 레이아웃이 깨지지 않는가.
153. **XSS 시크릿 스크립트**: 마크다운 텍스트 내 `<a href="javascript:alert('XSS')">` 악성 링크 방어 여부.
154. **거대 HTML 블록**: 사용자 프롬프트에 수천 줄의 HTML 코드가 있을 때 DOM 파싱 엔진이 멈추지 않는가.
155. **중첩 리스트 번호**: 1, 2, 3 이후 들여쓰기로 a, b, c가 나오는 복잡한 리스트가 올바르게 렌더링되는가.
156. **취소선 및 볼드체 혼합**: `~~**텍스트**~~` 와 같이 여러 마크다운 문법이 겹칠 때의 올바른 중첩 렌더링.
157. **줄바꿈 규칙(Hard Break)**: 마크다운 끝에 스페이스 두 번 없이 그냥 엔터만 쳤을 때 GFM 플러그인 설정에 따라 어떻게 렌더링되는가.
158. **수식 내 변수**: 인라인 수식 `$x = y^2$` 렌더링 시 주변 텍스트와 줄 간격(Line-height)이 어긋나지 않는가.
159. **빈 코드 블록**: ````javascript\n\n```` 처럼 내용이 없는 코드 블록이 렌더링되었을 때의 시각적 처리.
160. **테이블 오버플로우**: 열(Column)이 20개 이상인 거대 마크다운 테이블이 부모 영역을 넘어설 때 가로 스크롤이 생기는가.

## [그룹 17: 테마 및 접근성 전환 (Theme & A11y Switches)] (161-170)
161. **다크/라이트 토글 지연**: 테마 토글 시 `MessageList` 내부의 캔버스 렌더러나 iframe 기반 다이어그램이 즉시 테마를 따라가는가.
162. **시스템 테마 감지**: OS 설정이 변경될 때 `ThemeMode = auto` 상태에서 즉각적인 테마 반영.
163. **대비율(Contrast Ratio)**: 다크모드에서 인라인 코드(`<code>`) 배경색과 텍스트의 대비율이 WCAG 표준을 만족하는가.
164. **폰트 사이즈 동적 대응**: OS/브라우저 레벨에서 폰트 크기 150% 설정 시 가상화의 `estimateMessageHeight` 로직이 폰트 크기를 반영하는가.
165. **모션 축소(Reduced Motion)**: `prefers-reduced-motion` 켜짐 시 스크롤 점프의 `smooth` 속성이 `auto(즉시 이동)`로 대체되는가.
166. **로딩 애니메이션 접근성**: 스트리밍 중일 때 시각 장애인용 스크린 리더가 "응답 생성 중"임을 읽어주는가.
167. **언어 속성 변경**: 앱 내 언어 설정이 바뀔 때 마크다운 파서의 로케일 종속 규칙(날짜 등)이 즉시 갱신되는가.
168. **컬러 블라인드 모드**: 색각 이상 사용자를 고려하여 차트 생성 결과나 상태 배지가 색상 외에 아이콘/텍스트로도 구분되는가.
169. **줌 아웃 레이아웃**: 브라우저 배율을 50%로 극단적으로 줄였을 때 플렉스 박스가 무너지지 않고 정중앙을 유지하는가.
170. **프린트 미디어 쿼리**: `Ctrl+P` 시 `MessageList` 내부의 가상화 리스트가 일시적으로 해제되어 모든 메시지가 인쇄되도록(또는 최소한 경고라도) 처리되어 있는가.

## [그룹 18: 사용자 인터랙션 중첩 (Interaction Overlaps)] (171-180)
171. **복사 중 스크롤**: 텍스트를 드래그하여 블록을 씌운 상태에서 마우스 휠을 굴려 가상화 영역 밖으로 나갈 때 선택이 유지/해제되는 동작.
172. **모달 위 클릭 통과**: `SourceDrawer`가 떠 있을 때 그 바깥의 `MessageList` 본문을 클릭 시 Drawer가 닫히면서 클릭 이벤트는 소비(consume)되는가.
173. **스크롤바 드래그**: 커스텀 스크롤바를 마우스로 직접 잡고 위아래로 미친 듯이 흔들 때 가상화 렌더러가 비정상 종료를 일으키지 않는가.
174. **터치 앤 홀드(Touch & Hold)**: 모바일 환경에서 메시지를 꾹 눌렀을 때 네이티브 텍스트 선택 툴과 앱 자체의 팝오버 메뉴 간의 충돌.
175. **멀티 클릭 디바운싱**: 링크나 버튼을 의도적으로 초당 10회 이상 연타했을 때 불필요한 네트워크/상태 변경 요청이 블록되는가.
176. **스트리밍 중 복사**: 한창 생성 중인 텍스트 블록을 사용자가 `Ctrl+C` 할 때 지금까지의 텍스트가 정상 복사되는가.
177. **호버 잔상 현상**: 마우스를 빠르게 움직여 컴포넌트를 이탈할 때 버튼의 Hover 스타일이 영구적으로 남는(Sticky Hover) 문제 확인.
178. **초점(Focus) 상실 방지**: 키보드 탭 이동 중 가상화 영역 밖으로 스크롤되며 요소가 DOM에서 삭제될 때 포커스가 `body`로 튀지 않는가.
179. **입력창 동시 사용**: 스트리밍 진행 중에 사용자가 하단 Input 창에 새 질문을 치기 시작하면 `AutoBottom` 로직이 입력 경험을 방해하지 않는가.
180. **제스처 내비게이션 방해**: 맥북 트랙패드 두 손가락 스와이프(뒤로가기) 제스처가 마크다운 코드 블록 내의 가로 스크롤과 충돌하지 않는가.

## [그룹 19: 출처 카드 및 외부 리소스 로딩 (Citations & Resources)] (181-190)
181. **출처 로딩 지연**: `sourcePreviewLoading`이 매우 길어질 때 카드가 플레이스홀더 상태에서 깨지지 않고 스피너 등을 표시하는가.
182. **깨진 출처 이미지**: 출처의 `iconUrl`이나 썸네일 이미지가 404 에러일 때 텍스트 기반의 Fallback 아이콘을 보여주는가.
183. **다수 출처 동시 요청**: 한 메시지에 출처가 30개 있을 때 브라우저 동시 연결 수 제한(보통 6개)을 고려한 네트워크 병목 방지 로직.
184. **출처 캐시 유효성**: `sourcePreviewCache`에 저장된 데이터가 다른 대화방이나 재렌더링 시에도 적절히 재사용되는가.
185. **오프라인 상태의 출처**: 인터넷이 끊긴 상태에서 캐시되지 않은 출처 링크를 렌더링할 때의 UI 응답.
186. **출처 텍스트 오버플로우**: 출처 제목(`title`)이 200자 이상으로 매우 길 때 레이아웃을 밀어내지 않고 줄임표 처리되는가.
187. **출처 토큰 파편화**: 스트리밍 중 `\ue200`과 `cite\ue202`가 서로 다른 패치 청크로 쪼개져 도착했을 때 렌더러가 이를 대기/조립하는 기능(또는 방어 로직).
188. **내부/외부 링크 구분**: 출처가 아닌 마크다운 기본 링크(`[Google](http...)`)와 시스템 내부 링크(파일, 샌드박스 등)의 시각적 및 기능적 구분.
189. **보안 출처 차단**: `file://`이나 로컬호스트 주소를 출처로 지정하는 악의적 요청 렌더링 차단.
190. **출처 서랍(Drawer) 스크롤**: `SourceDrawer` 내에 출처 항목이 매우 많을 때 내부 스크롤이 작동하며, 메인 스크롤과 이벤트가 겹치지 않는가.

## [그룹 20: 극한 환경 및 디바이스 특성 (Extreme Conditions & Devices)] (191-200)
191. **10,000개 이상의 텍스트 줄**: 단일 메시지 내에 엔터가 1만 개 이상 있는 무의미한 폭탄 데이터 렌더링 시 브라우저 방어.
192. **초당 100회 이상의 상태 업데이트**: 메인 프로세스에서 `__GPT_RAW_PATCH__`가 버그로 인해 1ms마다 폭주할 때 리액트 상태 업데이트 스로틀링 여부.
193. **메모리 부족(OOM) 직전 환경**: 브라우저 탭 메모리가 1GB를 넘어 크래시 직전일 때 가상화 리스트가 선제적으로 렌더링 노드를 줄이는 최적화.
194. **초고해상도 다중 모니터**: 8K 모니터 두 개를 걸쳐서 창을 띄우고 스크롤할 때의 렌더링 프레임 속도(fps) 방어.
195. **낡은 브라우저 엔진 호환성**: ResizeObserver나 CSS `color-mix` 등 모던 API를 지원하지 않는 환경에서의 크래시 방지 및 폴백 UI.
196. **하드웨어 가속 강제 비활성화**: 사용자가 GPU 가속을 껐을 때(소프트웨어 렌더링) 무거운 스크롤 애니메이션이 유발하는 성능 저하 점검.
197. **네트워크 디버깅 모드(3G Throttle)**: 의도적으로 초당 50KB로 네트워크를 제한했을 때 이미지와 출처 데이터가 스크롤을 얼마나 지연시키는가.
198. **저전력 모드(Battery Saver)**: OS가 저전력 모드에 돌입하여 타이머 해상도나 애니메이션 프레임이 제한될 때의 동작 신뢰성.
199. **장시간 백그라운드 방치**: 탭을 10시간 이상 백그라운드로 내렸다가 복귀했을 때 밀려있던 렌더링 작업이 폭주하지 않고 현재 상태로 조용히 동기화되는가.
200. **기기 절전모드(Sleep) 진입 및 해제**: 컴퓨터가 슬립 모드에 들어갔다 깨어났을 때 스트리밍 상태나 가상화 인덱스가 손상되지 않는가.
