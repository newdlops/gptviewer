# GPTViewer 핵심 기능 명세 (Feature Specification)

본 문서는 리팩토링 과정에서 반드시 보존되어야 하는 GPTViewer의 핵심 기능을 정의합니다. 모든 코드 변경 후에는 본 명세의 기능들이 정상 작동하는지 확인해야 합니다.

## 1. 대화 데이터 임포트 및 갱신 (Import & Refresh)
- **ChatGPT 공유 URL 임포트**: `https://chatgpt.com/share/...` 경로를 통해 대화 내용을 가져올 수 있어야 함.
- **직접 대화 URL 임포트**: 실제 채팅 URL(`https://chatgpt.com/c/...`)을 통해 전체 대화 이력을 가져올 수 있어야 함.
- **새로고침 (Refresh)**: 기존 대화의 변경사항이나 추가된 메시지를 수동/자동으로 동기화할 수 있어야 함.
- **Sentinel API Flow**: 메시지 전송 시 4단계(WebSocket -> Prepare -> Wakeup -> Conversation) 흐름이 정상적으로 수행되어야 함.

## 2. 실시간 스트리밍 및 메시지 UI (Streaming & Rendering)
- **실시간 텍스트 출력**: 서버에서 오는 SSE 델타 데이터를 본문에 즉시 반영해야 함.
- **인라인 출처(Citation) 변환**: 스트리밍 중 도착하는 `【n】` 또는 `cite...` 토큰을 클릭 가능한 카드 링크(`InlineAssistantLink`)로 실시간 변환해야 함.
- **자동 스크롤 추적 (Auto Bottom)**: 
    - 스트리밍 시작 시 자동으로 바닥으로 이동.
    - 사용자가 위로 스크롤하면 추적 중단 (화면 떨림 방지).
    - 다시 바닥에 도달하면 추적 재개.
- **마크다운 렌더링**: 코드 블록(언어 하이라이팅), 수식, 이미지 등이 올바르게 표시되어야 함.
- **목차(TOC) 패널**: 대화 내 질문 및 헤딩을 감지하여 좌측 탭을 통해 빠른 이동이 가능해야 함.

## 3. 워크스페이스 및 트리 관리 (Workspace Management)
- **트리 구조**: 폴더와 대화를 계층적으로 구성하고 드래그 앤 드롭으로 이동할 수 있어야 함.
- **상태 보존**: 앱 재시작 시 활성 대화, 폴더 확장 상태, 스크롤 위치 등이 유지되어야 함.
- **검색 및 필터링**: 워크스페이스 내 대화 제목 및 요약 내용을 기반으로 검색이 가능해야 함.

## 4. 외부 연동 (Integration)
- **Google Drive 동기화**: 워크스페이스 스냅샷을 구글 드라이브에 업로드/다운로드하여 여러 기기 간 상태를 맞출 수 있어야 함.
- **웹 검색 토글**: 메시지 전송 시 웹 검색(`system_hints: ["search"]`) 옵션을 켜고 끌 수 있어야 함.

## 5. 성능 및 안정성 (Stability)
- **가상화 리스트 (Virtualization)**: 대화 내용이 길어져도(수백 개의 메시지) 버벅임 없이 부드러운 스크롤이 유지되어야 함.
- **메모리 관리**: 대규모 대화 파싱 시 메인 프로세스가 중단되지 않아야 함.
- **무한 루프 방지**: 상태 업데이트(setState)가 연쇄적으로 발생하여 브라우저가 멈추는 현상이 없어야 함.
