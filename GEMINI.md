# Gemini Project Context

## IntelliJ MCP Configuration

- 이 프로젝트에는 여러 개의 프로젝트가 동시에 열려 있을 수 있는 환경입니다.
- **필수 지침:** `IntelliJ MCP` 도구를 호출할 때는 반드시 `projectPath` 파라미터에 현재 작업 중인 **워크스페이스의 루트 경로(현재 디렉토리 경로)**를 동적으로 파악하여 명시적으로 포함해야 합니다. 하드코딩된 경로를 사용하지 마십시오.
- **안정성 확보:** 
  - IntelliJ가 백그라운드에 있다가 복귀할 때 일시적으로 MCP 응답이 지연되거나 세션 오류가 발생할 수 있습니다.
  - MCP 도구 호출 시 가능하다면 항상 `timeout` 파라미터를 넉넉하게(예: `3600000` - 1시간) 설정하여 타임아웃을 방지하십시오.
  - 만약 "No exact project is specified" 또는 "Timeout" 오류가 발생하면, 즉시 다시 시도(Retry)하거나 `get_repositories` 등의 가벼운 도구로 세션을 재활성화한 후 원래 작업을 수행하십시오.
  - **백그라운드 절전(App Nap) 이슈:** 사용자가 IntelliJ를 벗어나 크롬 등 다른 앱을 사용할 때, macOS의 절전 기능(App Nap)이나 IDE 자체의 백그라운드 처리로 인해 MCP 서버 연결이 끊기거나 응답이 멈출 수 있습니다. 이로 인해 에러가 발생한 경우, 당황하지 말고 사용자에게 "IntelliJ가 백그라운드에 있어 연결이 지연된 것 같습니다"라고 안내한 뒤 연결 복구를 재시도(`get_repositories` 호출 등) 하십시오.
  - 타임아웃 이후 MCP 서버가 계속 먹통이 되는 현상(전체 프로젝트 영향)을 막기 위해, 한 번 타임아웃이 나면 반드시 `get_repositories`를 호출하여 연결을 초기화/확인하는 절차를 거치십시오.
  - **절대 금지:** MCP 서버 오류(타임아웃, 응답 없음 등)가 발생했다고 해서 **일반 Shell 명령어(`grep`, `sed`, `find` 등)나 다른 파일 읽기/쓰기 도구로 우회(Fallback)하지 마십시오.** 항상 연결을 복구한 뒤 **반드시 다시 MCP 도구(`mcp_intellij-server_*`)를 사용하여 작업을 완수**해야 합니다.
  - 도구 호출 시 항상 `projectPath`를 명시함으로써, IDE가 어떤 프로젝트 컨텍스트인지 즉각 인식하도록 강제합니다.
